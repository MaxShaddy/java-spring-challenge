Get a paginated list of users, sorted by id, asc. Allows to specify parameters:


Defaults: page=0; size=50
page - page number zero based
size - items per page
Examples
  • Pagination - Default
  • Pagination - Page: 1 Size: 5

Filter by any of the listed fields (contains).Parameters: case insensitive. Condition: contain, and.


Defaults: all param is null nothing filtered.
firstName - string.
lastName - string.
email - string.
roleName - string. Existing roles: ADMIN, MANAGER, USER, CEO .
Examples
  • Filtering - Default(no filters applied)
  • Filtering - firstName: bo roleName: MaN
  • Filtering - firstName: bo roleName: user

Sort by any of the listed fields, either ascending or descending.Parameters: case insensitive.


Defaults: all param can be null or ommited.
firstName [ASC|DESC] or [asc|desc]
lastName [ASC|DESC] or [asc|desc]
email [ASC|DESC] or [asc|desc]
Examples
  • Sorting - Default(sorted by Id - ASC)
  • Sorting - firstName: asc lastName: DESC
  • Sorting - firstName: desc lastName: zzz (last name not applied as wrong param value).

Put all API together.


Filtering params(as on previous example except param name begins with 'f'):
fFirstName - string.
fLastName - string.
fEmail - string.
fRoleName - string. Existing roles: ADMIN, MANAGER, USER, CEO .
Sortin params:
sFirstName [ASC|DESC] or [asc|desc]
sLastName [ASC|DESC] or [asc|desc]
sEmail [ASC|DESC] or [asc|desc]
Pagination params: page=0; size=50
page - page number zero based
size - items per page
Examples
  • Sorting, Paging, Filtering - Default(not applied)
  • Filter: fFirstName=bob Sorting - lastName=DESC
  • Filter: fRoleName=user Sorting - sLastName=DESC Page: size=5 page=1
